# 物品列表 Flex 自适应布局

**功能名称**: 物品列表布局优化
**PRD 版本**: v1.0
**创建日期**: 2026-07-23
**作者**: MiMoCode

## 背景与目标

### 1.1 背景

当前物品列表页的物品卡片采用 Grid `auto-fill` 布局，列宽固定为 6rem。当容器宽度不是 6rem 的整数倍时，右侧会出现不自然的留白区域，视觉上不够紧凑。需要改为 Flex 自适应排列，使卡片在任意容器宽度下都能自然填充。

### 1.2 目标

1. 物品列表卡片改为 Flex 自适应排列，消除右侧留白
2. 保持卡片左对齐、固定间距的视觉效果
3. 分组模式和非分组模式均生效

### 1.3 成功标准

- 物品列表在不同屏幕宽度下无异常留白
- 卡片排列方式与当前一致（左对齐、自动换行）
- 分组/非分组/筛选/排序/分页功能不受影响

## 用户分析

### 2.1 目标用户

浏览物品档案的管理员

### 2.2 用户场景

| 场景 | 用户角色 | 目标 | 痛点 |
|------|----------|------|------|
| 浏览物品列表 | 管理员 | 快速查看所有物品卡片 | 容器宽度不适配时右侧有空白 |
| 按类型分组查看 | 管理员 | 分组浏览物品 | 分组内卡片排列不紧凑 |

## 功能需求

### 3.1 功能概述

将物品列表的卡片容器从 Grid 固定列布局改为 Flex 自适应排列，卡片自动换行填充可用空间。

### 3.2 功能列表

#### 功能点 1: 物品卡片 Flex 排列
- **描述**: 物品卡片容器改为 Flex 布局，卡片按行排列，超出容器宽度时自动换行
- **用户价值**: 任意屏幕宽度下卡片紧凑排列，无异常留白
- **验收标准**:
  - [ ] 物品卡片左对齐排列
  - [ ] 卡片间距一致
  - [ ] 容器宽度变化时卡片自动换行

#### 功能点 2: 分组模式兼容
- **描述**: 分组模式下每个分组内的卡片同样采用 Flex 排列
- **用户价值**: 分组和非分组视图体验一致
- **验收标准**:
  - [ ] 分组内卡片排列方式与非分组一致
  - [ ] 分组标题、分页等功能不受影响

### 3.3 页面/界面描述

| 页面 | 描述 | 关键元素 |
|------|------|----------|
| 物品列表 | 物品卡片以 Flex 方式自适应排列 | 物品卡片（正方形 tile）、筛选栏、分组标题、分页 |

### 3.4 异常与边界情况

| 情况 | 预期行为 |
|------|---------|
| 极窄屏幕（移动端） | 卡片换行，每行 1-2 个 |
| 极宽屏幕（桌面端） | 卡片铺满一行 |
| 筛选结果为空 | 显示空状态提示，不受布局影响 |

## 非功能需求

### 4.1 性能要求

无额外性能影响，仅 CSS 类名变更

### 4.2 兼容性要求

无浏览器兼容性变化，Flex 布局为现有项目已广泛使用的技术

## 依赖与约束

### 5.1 依赖

- 物品卡片组件 `ItemTile` 保持现有宽度（xl = 6rem）不变

### 5.2 约束

- 仅改动物品列表页，武器/装备列表保持 Grid 布局不变
- 不引入新的 UI 文案，无 i18n 变更
